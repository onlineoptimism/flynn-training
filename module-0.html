Welcome to the AI Tools Family

Three tools, one Claude. Here's why they're different, which one you should reach for, and why it actually matters for how you work at OO.

What this course is — and isn't

This isn't a "here's how AI works" lecture. You don't need to know what a large language model is. You just need to know how to use these tools to do your actual job better.

We built this course because OO now has three different Claude-based tools, and it wasn't super obvious which one to use for what. That ambiguity slows people down. So: this course removes the ambiguity.

By the end, you'll have real experience with all three tools, department-specific walkthroughs that match your actual work, and a live project deployed to your own OO subdomain. It's hands-on, it's self-paced, and yes — you're going to actually build something. 🛠️

How long will this take? Modules 0–3 are reading + exercises, roughly 20–30 minutes each. Module 3.1 (the deployment module) takes about an hour your first time. Module 4 is however big you want to make your project.

The three tools — a New Orleans analogy

Think about three different ways to experience amazing New Orleans food. Same city, same ingredients, totally different experience:

☕
Claude Chat

Texting your foodie friend

"Hey, where should I eat on Magazine Street?" — quick question, instant answer, totally conversational. Great for one-off tasks, drafts, and thinking things through. You ask, it answers, you move on.

👨‍🍳
Claude Cowork

A private chef in your kitchen

They show up at your house, open your fridge, and cook a full multi-course meal with what's actually in there. Cowork opens your actual files, works through multi-step tasks, and remembers what it did last time.

🏗️
Claude Code

Building the restaurant

They're not cooking dinner — they're building the kitchen, writing the menu, and when they're done, customers can actually show up. Claude Code builds real things: tools, sites, scripts that get deployed and live at a URL.

Same Claude brain behind all three. Different interface, different context, different superpowers.

Who should use what — and when

Use Claude Chat when...

  • You need a quick answer, draft, or idea right now
  • You're working from memory or a pasted document
  • You need it from your phone or without the desktop app
  • The task fits in one or two back-and-forth messages

Use Claude Cowork when...

  • The task involves files already on your computer (CSV exports, folders of docs, images)
  • You need Claude to do several things in sequence — analyze, then write, then organize
  • You want to walk away and come back to a finished result
  • You're doing a department-specific workflow (content gap analysis, batch captions, Google Ads review)

Use Claude Code when...

  • You want to build something real: a tool, a page, a script
  • You need the result to actually go live at a URL
  • You're working on something that spans multiple files and sessions
  • You want GitHub integration and version history
