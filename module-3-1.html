Module 3.1

GitHub + Cloudflare Deploy

You built something with Claude Code. Now let's put it on the internet — on your own OO subdomain. This guide walks you through every step, even if you've never touched GitHub before.

3.2 — What is GitHub?

GitHub is where code lives on the internet. Think of it like Google Drive, but specifically for code files. It tracks every change ever made, lets multiple people work on the same project without overwriting each other, and makes it easy to publish things to the web.

You don't need to know how to code to use GitHub for deployment. All you need is a free account and the ability to follow steps — which you clearly have, since you're here.

OO GitHub Org: All OO projects live under the onlineoptimism GitHub organization. You'll need to accept your invite before you can push projects there.

3.2 — What is Cloudflare Pages?

Cloudflare Pages is a free hosting service that watches your GitHub repo and automatically publishes your site every time you push new code. It's fast, reliable, and handles the technical hosting side so you never have to think about servers.

Once connected, your site goes live in under a minute after every update. OO uses Cloudflare Pages for all internal subdomains — including this course right now.

Free tier is plenty: Cloudflare Pages' free plan handles unlimited sites, unlimited bandwidth, and 500 deploys per month. You won't hit those limits.

3.3 — Step-by-Step Deployment

  1. Accept your OO GitHub invite Check your email for an invitation to the onlineoptimism GitHub org. Accept it. If you don't see one, ask in #ai-tools on Slack.
  2. Create a new GitHub repository Go to github.com, click the + icon, and select "New repository." Name it something like client-projectname. Set it to Public, skip the README checkbox, and click Create.
  3. Build your page with Claude Code Open Claude Code, point it at your project folder, and ask it to build your page. A simple starting prompt:
    Build a single-page HTML site for [Client Name]. Include a hero, about section, and contact CTA. Brand colors: [hex]. Save as index.html.
    When Claude is done, you'll have an index.html (and maybe a styles.css) in your folder.
  4. Push your files to GitHub In Claude Code, ask it to initialize a git repo and push to your new GitHub repo URL:
    Initialize a git repo here, add all files, commit with message "initial build", and push to https://github.com/onlineoptimism/your-repo-name.git
    Claude will run the git commands for you.
  5. Connect to Cloudflare Pages Log in at pages.cloudflare.com with your OO Cloudflare account. Click "Create application" then "Connect to Git." Select your GitHub repo, leave all build settings blank (static HTML needs none), and click Deploy.
  6. Add a custom CNAME for the OO subdomain Once deployed, go to your Cloudflare Pages project settings. Under "Custom domains," add your subdomain (e.g., projectname.onlineoptimism.com). Cloudflare will add the CNAME record automatically if your domain is on Cloudflare DNS — which ours is.
  7. Verify it's live Visit your subdomain URL in a browser. If it loads, you're done. If not, wait 2 minutes for DNS to propagate and try again. Then post in Slack to share the win. 🎉

3.4 — OO Subdomain Convention + Slack Template

All OO-deployed subdomains follow the same naming pattern:

clientname.onlineoptimism.com
projectname-clientname.onlineoptimism.com

Keep names lowercase, hyphenated, and short. No underscores. If you're not sure what to name it, ask your team lead before deploying.

Once your site is live, post in #ai-tools on Slack using this template:

🚀 Just deployed [project name] for [client name]! Live at: [your-subdomain].onlineoptimism.com Built with: Claude Code + Cloudflare Pages Repo: github.com/onlineoptimism/[repo-name] [Optional: one sentence about what it is or what it does]
