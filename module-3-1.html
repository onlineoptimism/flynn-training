Module 6

GitHub + Cloudflare Deploy

You built something with Claude Code. Now let's put it on the internet — on your own OO subdomain, protected so only @onlineoptimism.com accounts can access it.

6.1 — GitHub Setup

GitHub is where your code lives. Think of it like Google Drive for project files — it tracks every change, keeps a full history, and makes it easy to publish to the web. You don't need to know how to code to use it.

  1. Accept your OO GitHub invite Go to github.com/online-optimism and follow the on-screen prompts to accept your org invite. No invite yet? Ask in #ai-and-automations on Slack.
  2. Create your repo — and check "Add a README" When creating a new repo on GitHub, check the Add a README file box before clicking Create. This automatically creates a main branch, which is what Cloudflare Pages expects to watch. If you skip this, your repo starts empty with no branch and Cloudflare won't know what to deploy.
  3. Configure your new repository settings Click Settings at the top of your repo and make these changes:
    • Under Pull Requests: check "Allow Auto-Merge" and "Automatically delete head branches."
    • Under Issues: check "Auto-close issues with merged linked pull requests."
  4. Install Claude on the repo In your repo settings, find the Claude app integration and install it. This lets Claude Code push to your repo directly.
The right workflow: Cloudflare watches main and redeploys every time it gets new commits. Do your work on a separate branch, then merge a pull request into main when you're ready to go live. This way you never accidentally publish something half-finished.

6.2 — Deploy on Cloudflare Pages

Cloudflare Pages watches your GitHub repo and automatically publishes your site every time you push new code. Setup takes about 5 minutes.

  1. Open Workers & Pages In dash.cloudflare.com, use the left sidebar to navigate to Build > Compute > Workers & Pages.
  2. Create a new deployment Click New Deployment in the top right, then click Create Application.
  3. Connect to GitHub Click Continue with GitHub, then select your repository and click Next.
  4. Confirm the branch is main On the settings screen, check that the Production Branch field says main. If it shows a different branch name (e.g. a feature branch), change it to main now — this is the branch Cloudflare will watch and redeploy from on every merge.
  5. Deploy Leave all other settings as-is and click Deploy. Wait about a minute while it builds. If it fails, copy the error into Claude Code and do what it says.
  6. Add your custom subdomain Once deployed, click Domains > Add a custom domain > Add > Custom domain. Enter your subdomain (e.g., flynntest.onlineoptimism.com), wait for it to be accepted, and click Add Domain.
  7. Verify it's live Wait about 10 seconds, then visit your subdomain in a browser. If it loads, great. Now go lock it down with Zero Trust below.
Naming convention: Keep subdomains lowercase and hyphenated. Use yourname-test.onlineoptimism.com for experiments, clientname-tool.onlineoptimism.com for client projects.

6.3 — Zero Trust Access Control

Your page is live — but right now anyone with the URL can see it. This step locks it down so only @onlineoptimism.com email addresses can log in.

  1. Open Zero Trust In the Cloudflare sidebar, scroll down and click Zero Trust.
  2. Go to Applications Click Access Controls > Applications, then click Add an application.
  3. Choose Self-Hosted Click Self-Hosted. For Application Name, enter just the subdomain part (e.g., flynntest). Change Session duration to 1 month.
  4. Add the public hostname Click Add public hostname. Enter your subdomain in the first field and onlineoptimism.com as the domain.
  5. Attach the OO access policy Click Select existing policies, check off "For Optimists", and click Confirm.
  6. Save Click Next, scroll down, click Next again, then click Save.
Don't skip this step. Any page you deploy to an OO subdomain should have Zero Trust enabled before you share the URL. Takes 2 minutes and keeps client and internal content properly protected.
